Addendum

Game Process

  • Understanding Lights Out Game Rules: I have thorough understanding of the Lights Out Game rules was essential. The game involves a grid of lights that can be turned on or off. Clicking a light toggles its state and the state of adjacent lights.

  • Choosing a Representation: The decision to represent the game board using a 2D array was made. Each cell in the array corresponds to a light on the game board, simplifying the tracking of on/off states.

  • Implementing Game Logic: Functions like toggleLights, makeMove, and checkGameOutcome were implemented to handle game logic. Toggling lights, checking for wins, and tracking moves were crucial aspects of the logic.

  • Visual Design Considerations: CSS styling was applied to enhance the visual appeal of the game. Careful consideration was given to the background image, fonts, and colors to create an engaging and aesthetically pleasing design.

  • Responsive Design: Responsive design techniques were applied to ensure the game is visually appealing on various devices and screen sizes.

  • Random Solvable Initial Configuration: Generating a random solvable initial configuration was a challenge. Simulating random clicks to create the starting state required careful consideration to ensure the puzzle was always solvable.

  • Debugging and Optimization: Debugging the toggleLights function was crucial to maintaining the game's logic. The code was optimized for readability and maintainability.

  • Final Testing: The final implementation was thoroughly tested to ensure a smooth and bug-free gameplay experience. Testing involved checking for win conditions, maximum moves, and responsiveness.

HTML Structure

The game is structured using HTML to create a container for the game, a heading, the game board, and an information section. Buttons and links are used for user interaction, providing a clean and organized layout.

CSS Styling

The game's visual appeal is enhanced using CSS. The background image, fonts, and colors are carefully chosen to create an aesthetically pleasing experience. Responsive design techniques are applied to ensure the game looks good on different devices.

JavaScript Implementation

JavaScript is used to dynamically create the game board, handle user clicks, and update the game state. Functions like toggleLights, checkGameOutcome, and resetBoard are implemented to ensure smooth gameplay. The code is organized, commented, and follows best practices for readability and maintainability.

JavaScript Functions

  • initializeBoard(): This function sets up the initial game board by dynamically creating cells. Each cell is assigned a click event listener, linking it to the makeMove function.

  • makeMove(row, col): Handles player moves. Checks if the game is won or the maximum moves are reached. Toggles lights and updates the game state accordingly.

  • toggleLights(row, col): Toggles the state of the clicked cell and adjacent cells. Updates the board to reflect the changes.

  • updateBoard(): Updates the visual representation of the game board based on the current state of the lightsOutArray.

  • checkGameOutcome(): Checks if the game is won or if the maximum moves are reached. Displays alerts accordingly.

  • checkWin(): Checks if the game is won by verifying if all lights are turned off.

  • resetBoard(): Resets the game board by reloading the page. Used for starting a new game.

  • generateLightsOutArray(size): Generates a 2D array representing the game board with random initial light configurations. Ensures the game starts in a solvable state.

  • updateMoveCount(): Updates the displayed count of remaining moves on the game board.

  • Random Initialization: Randomly initializes the game board by simulating multiple clicks to provide a solvable initial configuration.

  • Event Listener for New Game Button: Adds an event listener to the "New Game" button for calling the resetBoard function when clicked.
